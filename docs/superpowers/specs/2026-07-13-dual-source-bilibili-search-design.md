# 双来源转换与 B 站搜索设计

## 目标

在不修改 SoulX-Singer 上游的前提下，扩展 SVCVC-API-SVF 的现有转换页面和 `/convert` API：目标歌曲与参考音色均可从文本来源或本地上传取得；文本来源支持 B 站、网易云和原有的 HTTP 音频链接。新增独立的歌曲搜索页，提供 B 站音乐分区和网易云搜索结果，方便复制链接回填到任一来源输入框。

本设计保持 SoulX 原有的一体化工作方式：SoulX 自行做人声分离、F0、音色转换和可选的自动伴奏混音。本项目不会请求或依赖克隆人声/原伴奏分轨接口。

## 范围

### 转换页

- 将原有 `song_name_src` 标注为“目标歌曲来源”，支持 B 站 BV 号或视频链接、网易云链接或 ID、既有 HTTP(S) 音频链接和本地路径。
- 为目标歌曲增加可选本地文件上传；上传文件优先于目标来源文本。
- 增加“参考音色来源”文本框和可选本地文件上传，支持与目标歌曲相同的来源类型。
- 保留 `voice_profiles` 下拉框，作为已保存参考音色的兼容备用选择。
- 参考音色解析优先级固定为：参考上传文件、参考来源文本、已保存音色下拉框。三者均未提供时任务在开始前报错。
- 当参考来源是歌曲或视频时，默认沿用现有 `prompt_vocal_sep` 设置以分离参考人声；已保存 profile 继续读取它的 `prompt_vocal_sep` 元数据或现有默认值。
- 延续现有自动变调、手动升降调、自动混伴奏、步数、CFG、种子、缓存和 MP3 导出行为。

### API 兼容性

`/convert` 现有 11 个位置参数和顺序保持不变：

```text
song_name_src, model_dropdown,
prompt_vocal_sep, target_vocal_sep, auto_shift, auto_mix_acc,
pitch_shift, n_step, cfg, seed, random_seed
```

新参数只追加在末尾，且都具有默认值：`target_upload`、`reference_source`、`reference_upload`。因此现有 AstrBot 插件仍会按原方式把歌曲传给 `song_name_src`、将 profile ID 传给 `model_dropdown`，无需升级即可继续工作。

### B 站来源

- 新建独立的 B 站来源模块，负责识别 `BV...`、`b23.tv` 短链和 `bilibili.com/video/...` 链接，并统一为可由既有下载器处理的视频链接。
- 对关键词不做隐式搜索：转换输入框只接受明确链接/BV 号，避免将普通文本误判并触发风控。关键词通过搜索标签页显式检索。
- 下载失败、地区限制、需要登录、风控或没有音频流时，返回带来源名称的可读错误，不回退成错误的网易云或 HTTP 下载。

### 搜索页

- 新增“歌曲搜索”标签页，与转换页并列。
- 提供来源选择：B 站和网易云；B 站搜索固定使用音乐分区。
- 搜索结果包含封面、标题、作者/UP 主和可用于转换输入框的完整链接。
- 每一项结果提供复制链接按钮；页面同时提供当前选中结果的链接文本，便于无法使用剪贴板权限的环境手动复制。
- 搜索仅用于发现链接，不会自动下载、不会自动填入转换表单，也不会自动启动转换。

## 实现边界

1. 将来源解析与下载缓存从 `app.py` 中抽出为小模块；目标和参考共用同一个安全校验、下载、文件类型和大小限制逻辑。
2. 新增歌曲搜索模块，分别封装 B 站音乐分区查询和网易云查询，返回统一的 `title`、`artist`、`cover`、`url` 数据结构。
3. `convert` 在解析目标和参考后继续调用现有 SoulX 普通转换端点；当参考来自临时下载/上传文件时，调用完成后清理临时目录，但不删除缓存中的可复用下载文件。
4. 不修改 SoulX-Singer-AMD-Patch，不增加 `/cover`，不增加独立人声/伴奏音轨、混响、回声或独立音量控制。

## 错误处理

- 无目标歌曲时明确提示需要目标来源或上传文件。
- 无参考音色时明确提示填写参考来源、上传参考文件或选择已保存音色。
- B 站关键词搜索无结果与 B 站请求被拒绝分别报告，避免将风控误说成“没有视频”。
- 封面加载失败不影响搜索结果的标题、链接或复制功能。
- 已保存 profile 不存在时，保留现有的明确错误；若同时指定了更高优先级的参考上传/来源，则不读取 profile。

## 验证

- 单元测试覆盖来源识别、优先级、旧 `/convert` 的 11 参数签名、B 站链接解析、搜索结果规范化和错误消息。
- 使用伪造 SoulX 客户端验证：参考来源解析出的路径会作为普通 prompt 传入现有转换调用，而不会调用任何分轨端点。
- 手动验证转换页中的旧 profile 工作流仍能完成一次固定种子转换；再分别使用 B 站目标链接和 B 站参考链接完成转换。
- 手动验证 B 站和网易云搜索都能显示链接并复制；B 站封面请求失败时页面仍可使用。
